에러 로그 시스템은 애플리케이션의 버그와 문제를 신속하게 파악하고 대응하기 위해 필수적입니다.
에러 로그 구축 핵심 단계
로그 레벨 설정: DEBUG, INFO, WARN, ERROR 등으로 구분하여 중요도에 따라 기록.
로깅 프레임워크 적용:
알림 시스템 연동: 에러 발생 시 메일 등으로 즉시 통보하여 대응 시간 단축.
로그 내용 포함 항목: 타임스탬프, 에러 수준, 에러 메시지, 스택 트레이스(어디서 발생했는지), 사용자 정보 등.
구축 시 고려사항
운영/개발 환경 분리: 운영 환경(Production)에서는 디버그 로그를 최소화하고 필수 에러만 기록.(WARN 이상부터)
그리고 얘네를 /admin/dashboard에서 에러 로그 확인이라는 칸을 만들어서
admin에서 테이블 리스트로 확인할 수 있고 (WARN, ERROR만 띄우고 날짜, 유형으로 정렬 및 필터링할 수 있게)
그리고 클릭하면 상세 내역이 보이게 해줘
